# Build Admin course content screens

Build the Admin-facing LiveView screens for managing Course, Chapter, and Lesson content through basic web flows. This should expose the content lifecycle and ordering behavior in a usable MVP interface.

Decisions baked in from the PRD:
- Content is ordered by `created_at` for the MVP.
- Manual ordering controls and drag-and-drop ordering are out of scope for the MVP.
- Publish does not require a confirmation dialog for MVP.
- Archive requires confirmation because it hides the Course from new Learners.
- Empty-state and validation copy can be simple implementation copy for MVP.

## Acceptance Criteria

- [ ] An Admin can access Admin screens.
- [ ] Non-Admins are blocked from Admin screens.
- [ ] Admin Course list has an empty state with a Create Course action.
- [ ] Admins can create and edit Courses.
- [ ] Admins see validation messages when Course title or short description is missing.
- [ ] Admins can create and edit Chapters within Courses.
- [ ] Admins see validation messages when Chapter title is missing.
- [ ] Admins can create and edit Lessons within Chapters.
- [ ] Admins see validation messages when Lesson title or rich text body is missing.
- [ ] Courses, Chapters, and Lessons are shown by `created_at` order.
- [ ] Admins can Publish only structurally valid Courses.
- [ ] Admins can Archive Published Courses with confirmation.
- [ ] Admins can delete Draft Courses, Chapters, and Lessons through the UI.
- [ ] Admins are offered Archive instead of deletion for Published Courses.
- [ ] Protected Chapter or Lesson deletion redirects with a flash explaining that Published or Archived Course content cannot be deleted.
- [ ] Admin screens have automated coverage for core create/edit/order/publish/archive flows.
